# タイトル: <実装タスクの名称>

作成日: YYYY-MM-DD

## 概要

このタスクの目的と背景を簡潔に記述します。

- **目的**: このタスクで達成したいこと
- **背景**: なぜこのタスクが必要なのか
- **スコープ**: このタスクの範囲（含むもの/含まないもの）
- **参照**: 関連する Issue、PR、ドキュメントへのリンク

## 方針

### アプローチ

このタスクを実装するための基本的なアプローチや戦略を記述します。

- どのような技術や手法を使うか
- どのような順序で進めるか
- どのような制約や前提条件があるか

### 設計方針

実装における重要な設計判断や原則を記述します。

- アーキテクチャの選択理由
- パターンやベストプラクティスの適用
- トレードオフと選択理由

### 品質基準

タスク完了の定義と品質基準を明確にします。

- テストカバレッジの目標
- パフォーマンス要件
- コードレビュー基準
- ドキュメント要件

## タスク

実装すべき具体的なタスクをチェックリスト形式で列挙します。
フェーズやカテゴリに分けて整理すると管理しやすくなります。

**重要**: すべてのタスクは `- [ ]` (未完了) または `- [x]` (完了) の形式で記述し、完了状況を明示してください。

### Phase 1: <フェーズ名>

- [ ] タスク 1
  - [ ] サブタスク 1-1
  - [ ] サブタスク 1-2
- [ ] タスク 2
- [ ] タスク 3

### Phase 2: <フェーズ名>

- [ ] タスク 4
- [ ] タスク 5

### Phase N: <最終確認>

- [ ] コード整形（`cargo fmt`）
- [ ] Lint チェック（`cargo clippy`）
- [ ] テスト実行（`cargo test`）
- [ ] ドキュメント更新

## 進捗状況

タスクの進行状況を記録します。

- **開始日**: YYYY-MM-DD
- **現在のフェーズ**: Phase N
- **完了したタスク**: X/Y
- **次のアクション**: 次に行うべき具体的なタスク
- **ブロッカー**: 進行を妨げている問題（あれば）

## 備考

実装時の注意事項や重要な情報を記述します。

- コーディング規約
- テストフォーマット
- 命名規則
- その他の制約や注意点

## 参考資料

- 関連ドキュメントへのリンク
- 参考にした記事や書籍
- 技術的な背景情報

---

## このテンプレートの使い方

1. **タイトル**: `# タイトル: ` の後に実装タスクの名称を記述
2. **作成日**: 実際の作成日（YYYY-MM-DD 形式）に置き換え
3. **概要**: 目的、背景、スコープ、参照を埋める
4. **方針**: アプローチ、設計方針、品質基準を具体的に記述
5. **タスク**: 実装すべき内容をチェックリスト形式で列挙
6. **進捗状況**: 実装の進行に応じて更新
7. **備考**: プロジェクト固有のルールや注意事項を記述
8. **参考資料**: 必要に応じて追加

### 必須セクション

最低限、以下のセクションは必ず含めてください：

- タイトル
- 概要
- 方針
- タスク

その他のセクション（進捗状況、備考、参考資料）は必要に応じて追加・削除してください。
